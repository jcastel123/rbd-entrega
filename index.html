Sección 1 - AWS

Servicios AWS

  • EC2: Servicio de máquinas virtuales en la nube.
  • RDS: Bases de datos gestionadas por AWS.
  • S3: Almacenamiento de archivos.
  • EBS: Discos virtuales para EC2.
  • DynamoDB: Base de datos NoSQL.
  • CloudFront: CDN para acelerar webs.
  • Lambda: Ejecución de código sin servidores.
  • CloudWatch: Monitorización.
  • IAM: Usuarios y permisos.
  • Route53: DNS de AWS.

Arquitectura AWS

Los servicios de datos se ubican en redes privadas (RDS, EBS) para evitar accesos externos. Los servicios públicos como EC2 y CloudFront se sitúan en redes públicas. La arquitectura se despliega en dos CPDs dentro de una misma región para garantizar alta disponibilidad.

Sección 2 - Primer servidor web en AWS

Se creó una instancia EC2 con Apache, protección contra terminación activada, página web Hello World, monitorización activa, grupo de seguridad con HTTP habilitado, reescalado de la instancia y prueba de protección contra terminación.

Sección 3 - AutoScaling y Load Balancer

Arquitectura formada por VPC, subredes públicas, instancias EC2, AutoScaling Group, Elastic Load Balancer y monitorización mediante CloudWatch.

Big Data

Las 5 Vs

  • Volume: Cantidad de datos generados
  • Velocity: Velocidad de generación
  • Variety: Tipos de datos
  • Veracity: Calidad de datos
  • Value: Valor obtenido de los datos

Ejemplos

  • Volume: Redes sociales con millones de posts
  • Velocity: Datos de tráfico GPS
  • Variety: Datos de clientes
  • Veracity: Corrección de duplicados
  • Value: Predicción de demanda

Limitaciones de BBDD tradicionales

  • No escalan fácilmente
  • No procesan datos en tiempo real
  • Solo datos estructurados
  • Validación lenta
  • Análisis complejo

Hadoop

Cloudera: HDFS, YARN, MapReduce, Hive, Impala, Hue

Hortonworks: HDFS, YARN, MapReduce, Hive, Pig, Ambari, HBase

MapReduce

Modelo de procesamiento distribuido. Map genera pares clave-valor y Reduce agrupa y resume datos. Ejemplo: contar palabras en un libro.

Hive vs MySQL

MySQL es transaccional y orientado a aplicaciones. Hive es analítico y orientado a Big Data.

Tabla externa Hive

CREATE EXTERNAL TABLE IF NOT EXISTS sensor_data (
 sensor_id INT,
 temperature DOUBLE,
 city STRING,
 timestamp TIMESTAMP,
 is_raining BOOLEAN,
 humidity INT,
 wind_speed DOUBLE,
 extra_metrics STRING
)
ROW FORMAT SERDE 'org.apache.hadoop.hive.serde2.OpenCSVSerde'
WITH SERDEPROPERTIES (
 "separatorChar" = ",",
 "quoteChar" = "\"",
 "escapeChar" = "\\"
)
STORED AS TEXTFILE
LOCATION 's3://retoBigData/input/'
TBLPROPERTIES (
 "skip.header.line.count" = "1"
);

Apache Spark

Spark procesa datos en memoria, es más rápido que Hadoop MapReduce y soporta batch, streaming, machine learning y gráficos.

Modelo de captación de datos

  • sensor_id (Entero)
  • temperature (Decimal)
  • city (Texto)
  • timestamp (Fecha/Tiempo)
  • is_raining (Booleano)
  • humidity (Entero)
  • wind_speed (Decimal)
  • extra_metrics (Map)
